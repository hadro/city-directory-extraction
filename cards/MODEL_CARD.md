# city-directory-extractor (<SIZE>)   <!-- e.g. 0.8b / 2b / 4b -->

A small open model fine-tuned to turn **one line of a historical US city-directory entry**
into a structured record. It replaces the Gemini NER step of the `directory-pipeline`
project for the city-directory *persons* shape, and is meant to run cheaply at scale and
offline. One of a sibling family (<0.8b / 2b / 4b>).

## What it does

Input: a single (often OCR-noisy) directory line plus a tiny context tag.
Output: one **pipe-delimited** row with these fields, copied verbatim, empty when absent:

```
name|is_business|spouse_name|race_designation|occupation_role|employer|address|home_address
```

It is trained for two real "dialects": **Tulsa 1921** (Polk-Hoffhine; `r/b/rms` address
codes, separate `employer`, `(c)` marker) and **NYC Doggett/Trow 1850–1890** (separate
work/home address, `wid`/`col'd` markers).

## How to use

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

m = "<you>/city-directory-extractor-<SIZE>"
tok = AutoTokenizer.from_pretrained(m)
model = AutoModelForCausalLM.from_pretrained(m, torch_dtype="auto", device_map="auto")

FIELDS = ["name","is_business","spouse_name","race_designation",
          "occupation_role","employer","address","home_address"]
SYSTEM = ("You convert ONE line from a historical US city directory into structured fields. "
          "Output exactly one row of pipe-separated values in this order:\n" + "|".join(FIELDS) +
          "\nCopy values verbatim from the line. Use True/False for is_business. Leave a field "
          "empty (nothing between the pipes) when the line does not contain it. Output only the row.")

line = "Abernathy Frank H (Gertrude), boilermkr, r 303 N Boston av"
msgs = [{"role": "system", "content": SYSTEM},
        {"role": "user", "content": f"[dialect=tulsa; year=1921] {line}"}]
ids = tok.apply_chat_template(msgs, add_generation_prompt=True, return_tensors="pt").to(model.device)
out = model.generate(ids, max_new_tokens=128, do_sample=False)
row = tok.decode(out[0, ids.shape[1]:], skip_special_tokens=True).strip()
record = dict(zip(FIELDS, (row.split("|") + [""] * 8)[:8]))
```

The `[dialect=…; year=…]` tag carries page-level context that isn't in the line itself; pass
the dialect/year you're processing (`tulsa` or `nyc`).

## Limitations & out-of-scope

- **Line-level only.** Expects one entry per call; page segmentation is upstream.
- **Two dialects.** Other directories/eras/languages are out of distribution (see the FTD
  transfer numbers for how far it generalizes).
- **Verbatim, not normalized.** Copies values as printed (`av`, `r`, abbreviations); it does
  not expand or geocode them.
- **Silver in-domain labels.** Tulsa labels come from Gemini extraction unless hand-reviewed.

## Ethical considerations

The source directories encode the social categories of their era. The model reproduces, **verbatim
and without endorsement**, the directories' historical markers — `race_designation` (the
"colored"/`(c)`/`col'd` listing used under segregation) and widow status — because faithful
extraction is required for historical and genealogical research (e.g. studying Tulsa's
Greenwood district before the 1921 Race Massacre). Downstream users should handle these fields
with appropriate historical context and care, not as contemporary descriptors.
